# KANBAN BOARD / TICKETS - Phase 1: MVP Core & Real-time Text Chat

## 1. 데이터베이스 및 인프라 영역 (DB & Infra)

### 🎫 Ticket [DB-1]: Supabase 스키마 DDL 작성 및 테이블 생성
- **내용**: `schema.sql` 파일을 작성하고 Supabase SQL Editor를 통해 테이블(`profiles`, `spaces`, `categories`, `channels`, `members`, `messages`)을 생성합니다.
- **인수 기준 (Definition of Done)**:
  - 지정된 컬럼 및 외래키가 올바르게 설정되어 생성되어야 함.

### 🎫 Ticket [DB-2]: 신규 회원 가입용 프로필 생성 트리거 구현
- **내용**: Supabase `auth.users`에 신규 회원 생성 시 `public.profiles`에 자동으로 동일 `id` 레코드를 삽입하는 PostgreSQL trigger 및 function을 등록합니다.
- **인수 기준**:
  - 회원 가입 테스트 진행 시 `profiles` 테이블에 해당 UUID의 프로필 행이 정상 생성되어야 함.

### 🎫 Ticket [DB-3]: Row Level Security (RLS) 정책 활성화 및 설정
- **내용**: 스페이스 멤버십 기반으로 다른 사용자가 타인의 비공개 스페이스나 메시지를 조회하지 못하도록 테이블별 RLS Policy를 작성합니다.
- **인수 기준**:
  - 가입하지 않은 비멤버 계정으로 스페이스 및 메시지 조회 API 요청 시 권한 에러(403)가 반환되어야 함.

---

## 2. 상태 관리 및 비즈니스 로직 영역 (State & Logic)

### 🎫 Ticket [LOGIC-1]: Zustand Store 설계 및 구축
- **내용**: Next.js 클라이언트에서 전역으로 관리할 스페이스 목록, 채널 목록, 현재 활성화된 채널, 실시간 메시지 목록 상태 스토어를 구현합니다.
- **인수 기준**:
  - 컴포넌트 마운트 시 API 호출을 통해 전역 상태에 데이터가 적절히 로드되어야 함.

### 🎫 Ticket [LOGIC-2]: Supabase Realtime & Presence 연결 훅 구현
- **내용**: 특정 텍스트 채널에 진입했을 때 실시간 메시지 웹소켓 채널을 구독하고, 스페이스에 접속해 있는 유저들의 Presence 상태를 트래킹하여 Zustand 스토어에 동기화하는 커스텀 훅을 작성합니다.
- **인수 기준**:
  - 다른 기기로 테스트 접속 시 실시간으로 메시지가 전송되고 유저의 온라인/오프라인 상태 아이콘이 실시간 갱신되어야 함.

---

## 3. 사용자 인터페이스 영역 (UI & Client)

### 🎫 Ticket [UI-1]: 회원가입 및 로그인 페이지 구현
- **내용**: 이메일/비밀번호 방식을 지원하는 다크 모드 스타일의 인증 화면을 퍼블리싱하고 Supabase Auth와 바인딩합니다.
- **인수 기준**:
  - 가입/로그인 완료 시 메인 대시보드 스페이스 페이지로 리다이렉트되어야 함.

### 🎫 Ticket [UI-2]: 메인 레이아웃 퍼블리싱 (Sleek Premium Dark Mode)
- **내용**: 디스코드 레이아웃(좌측 스페이스 네비게이션, 중간 채널 카테고리 사이드바, 우측 접속자 목록, 중앙 채팅 영역)을 글래스모피즘이 적용된 다크 테마로 구현합니다.
- **인수 기준**:
  - 반응형 디자인으로 모바일 및 데스크톱 브라우저에서 레이아웃이 깨지지 않아야 함.

### 🎫 Ticket [UI-3]: 스페이스 생성 및 초대 링크 팝업 모달 구현
- **내용**: 새로운 스페이스를 이름과 아바타를 포함해 생성하는 폼을 구현하고, 초대 코드를 생성하여 클립보드에 복사할 수 있는 초대 관리 UI를 개발합니다.
- **인수 기준**:
  - 신규 생성 시 자동으로 '기본' 카테고리와 '# 일반' 텍스트 채널, '🔊 로비' 음성 채널 데이터가 함께 DB에 생성되어 UI에 즉시 동기화되어야 함.

### 🎫 Ticket [UI-4]: 실시간 채팅방 컴포넌트 구현
- **내용**: 메시지 목록 렌더링 영역, 메시지 전송 인풋 바, 스크롤 하단 자동 고정 기능이 포함된 채팅 UI를 연동합니다.
- **인수 기준**:
  - 메시지 전송 시 실시간으로 리스트가 누적되고, 하단으로 부드러운 모션과 함께 스크롤되어야 함.

# KANBAN BOARD / TICKETS - Phase 2: 실시간 음성/화상 통화 & 스테이지 채널

## 1. 백엔드 및 API 영역 (API & Integration)

### 🎫 Ticket [VOICE-1]: LiveKit 토큰 생성 API Route 구현
- **내용**: `src/app/api/livekit/token/route.ts` API 엔드포인트를 구현하여 클라이언트 요청 시 LiveKit 접속 토큰을 생성 및 반환합니다.
- **인수 기준**:
  - Supabase Auth를 통한 유저 세션이 존재하지 않거나, 요청한 채널의 스페이스 멤버가 아닐 경우 에러(401/403)를 반환해야 함.
  - 스테이지 채널 요청 시, ADMIN/OWNER 역할이 아니면 `canPublish` 권한이 `false`인 토큰이 발행되어야 함.

---

## 2. 클라이언트 및 상태 관리 영역 (Client & State)

### 🎫 Ticket [VOICE-2]: LiveKit HTML5 SDK 연동 및 Zustand 상태 확장
- **내용**: `livekit-client` SDK를 설치하고, 활성 음성 채널 입장/퇴장 상태 및 오디오/비디오 트랙 연결 관리를 위한 Zustand Store 액션을 구현합니다.
- **인수 기준**:
  - 채널 클릭 시 LiveKit SFU 룸에 접속되며, 마이크/비디오 스트림 공유가 준비되어야 함.

---

## 3. 사용자 인터페이스 영역 (UI & Components)

### 🎫 Ticket [VOICE-3]: 접이식 Collapsible Voice/Video 그리드 UI 구현
- **내용**: 중앙 채팅 패널 상단에 오디오/비디오 참여자들을 타일 형태로 렌더링하고 비디오 스트림을 스트리밍하는 접이식 그리드 UI를 퍼블리싱합니다.
- **인수 기준**:
  - 접기/펴기 버튼을 클릭했을 때 그리드 영역이 부드럽게 슬라이드 애니메이션과 함께 열리거나 닫혀야 함.
  - 화면 공유 시, 해당 스트림 타일이 다른 타일보다 넓게 강조 표시되어야 함.

### 🎫 Ticket [VOICE-4]: Stage Channel 전용 UI 권한 분기 및 마이크 상태 제어
- **내용**: 스테이지 채널 입장 시 일반 유저는 마이크 버튼이 비활성화되며 청취만 가능하고, 스페이스 ADMIN/OWNER 유저는 마이크 제어 및 방송(Publish)이 가능하도록 UI 제어 로직을 연동합니다.
- **인수 기준**:
  - 일반 MEMBER 계정으로 스테이지 채널 접속 시 오디오 송출이 불가능하고 청취 상태가 유지되어야 함.
